支持该 op 的集合按值升序 (快/低在上)，条长 ∝ 值 (log 刻度)，标值 + Big-O，金色高亮 = 最优。

对每个规范操作桶，自动取参考档位下最快的集合。派生口径：仅在可比的批量 / 随 n 缩放操作间比较，已排除单发操作 (Peek / Drain / SetItemOne / AddOne)。注意 Immutable 的 Remove/Set 仍是改/删单个元素返回新版本，请结合「代表 op」+ Big-O 判断。

需求场景推荐集合理由
该集合各 op × N 的 time / GC / Big-O。长板 = 它最快的 op，短板 = 最慢 / O(n) 的 op。
⚠ 读数警示
① 跨 N 原始 ms 非同口径：time_ms 是单次 measurement 耗时，内部迭代数随 N 缩放 → N=1/100/10000 的绝对 ms 不可直接横比；只在同一 (集合, 操作) 内沿 N 看趋势，或在同一 N 内横比集合。
② GC 字节粗粒度：分辨率约几十 KB；小档 / 碎片化可能读 0 = 未解析到 / 低于分辨率，非保证真零分配。
